### Step 1.4 — Wire peak-stats writes

`src/run/run-controller.ts` — at end of `finishRun`, after `analyzeRun(...)`, before `derivedCurveRepository.upsert`:

```ts
const peak = result.points.reduce((b, p) => p.wheel_power_kw > b.wheel_power_kw ? p : b, result.points[0]);
const peakTorque = result.points.reduce((b, p) => p.wheel_torque_nm > b.wheel_torque_nm ? p : b, result.points[0]);
if (peak) {
  await this.opts.runRepository.update(runId, {
    peak_power_kw: peak.wheel_power_kw,
    peak_torque_nm: peakTorque.wheel_torque_nm,
    peak_power_rpm: peak.rpm,
  });
}
```

`src/analysis/re-analyze.ts` — mirror the same logic so legacy runs get backfilled on next view.

## Phase 2 — Screens (parallel agents)

After Phase 1 commit, dispatch the following agents in parallel. Each agent receives:

1. A pointer to the spec (`docs/superpowers/specs/2026-05-26-v0.2-product-polish-design.md`)
2. The exact files they own (no other files to edit)
3. The new primitives they should use (units, formatters, GpsWarmupCard, etc.)
4. Specific requirements lifted from the spec

Files: `src/ui/auth/login-screen.tsx`, `src/ui/settings/settings-screen.tsx`

Login:
- Add `<BrandLogo size={48} />` centered
- Tagline below: "Your phone is a dyno."
- Replace blue button with `bg-amber-500 hover:bg-amber-400 text-zinc-950` matching app palette

- Section 4 — Permissions: unchanged (location, wake lock)
- Section 5 — Developer (collapsed by default visually but always visible): Replay mode (existing), link to `/recordings`
- Section 6 — About: unchanged

### Agent B — Garage + Vehicle Detail

Files: `src/ui/garage/garage-screen.tsx`, `src/ui/garage/vehicle-detail.tsx`

Garage:
- Vehicle card subline: `${count} runs · best ${formatPower(maxPeak)}` (loaded by fetching runs per vehicle on mount; cache result during render). If 0 runs: "No runs yet"
- Empty state (no vehicles): keep CTA, add a "How it works" card with 3 steps (Add vehicle / Calibrate gear / Drive and record). Each step is one line with a small numbered circle

Vehicle detail:
- Replace `started_at` ISO string with `formatRelativeTime(started_at)`
- Run rows show `formatPower(peak_power_kw)` (units-aware) when run is complete; "—" otherwise
- New header stats row (3 stat cards): Runs total / Best power / Last run (relative time)
- Stats card styling matches the existing zinc-900 cards

### Agent C — Run Review + PowerCurveChart

Files: `src/ui/run/run-review-screen.tsx`, `src/ui/components/power-curve-chart.tsx`

PowerCurveChart:
- New prop `mode?: 'power' | 'torque' | 'both'` (default 'power')
- New prop `palette?: string[]` (default the new amber-led palette)
- New prop `unit?: PowerUnit` for axis label and y-value conversion
- New prop `highlightLabel?: string` — adds `★ ` prefix and width 3 for matching series
- When `mode='both'`: two y-axes, power on left, torque on right (uplot `scale: 'power' | 'torque'`)

Review screen:
- 2x2 stat grid replacing the existing 1x2:
  - Peak power (primary: units-pref + secondary: opposite unit in muted text)
  - Peak torque + at-RPM
  - Power band (RPM range with ≥80% peak power, e.g. "3800 – 5600 RPM")
  - Δ vs best (e.g. "+5 kW vs your best"; "Personal best!" if it's the new best; "—" if first run)
- Above chart: `<SegmentedControl>` Power / Torque / Both
- Run title field above notes: pre-fill with `${gear_label} · ${formatShortDateTime(started_at)}` if `run.title` is empty
- Save persists `title` via `runRepository.update(runId, { title })` alongside notes
- Below notes, three buttons in a row: Save | Discard (existing), plus new buttons: Export CSV, Share image
- Export CSV: `rpm,wheel_power_kw,wheel_torque_nm\n...` downloaded as `dynorun-{title}.csv`
- Share image: render a 1080×1080 PNG with a `<canvas>` (manually drawn — see below), use Web Share API if available else download
  - Canvas composition: dark bg, brand logo top-left, vehicle name + title, peak stats grid, and a downsampled curve. Keep the render code in a new helper `src/app/share-image.ts`

To fetch previous best for Δ: `runRepository.listByVehicle(run.vehicle_id)`, filter `status==='complete' && id !== run.id`, find max `peak_power_kw`.

### Agent E — Live Run

Files: `src/ui/run/live-run-screen.tsx`

- Use `<GpsWarmupCard>` instead of inline warmup markup
- Read `localStorage['dynorun:countdown'] === 'true'`; if true, render `<CountdownOverlay>` on Start tap and call `controller.start()` at GO
- `pulseStart()` on actual start
- `pulseStop()` on stop (both auto-stop path via `onStateChange` transitioning to 'analyzing' and manual stop)
- Live peak power estimate during recording: maintain a 5-sample ring of `{t_ms, speed_mps, rpm}`; compute `dv/dt`, multiply by vehicle mass and speed (`F=ma`, `P=F*v`). Display under RPM (smaller font, "live peak: 142 kW")
  - The vehicle mass is reachable via `vehicleRepository.get(vehicle.id)` once during warmup
- "Recording" panel: power readout gets units-aware via `useUnits().format(estPeak)`

- RPM input: keep text field on top, add a stepper row underneath with `-250 | 3000 | +250` buttons. Stepper modifies the same state
- Layout: gear chip group first, RPM stepper second, instructional card stays at top

Measure step:
- Replace inline GPS warmup card with `<GpsWarmupCard>`
